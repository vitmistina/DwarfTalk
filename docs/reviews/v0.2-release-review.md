## 4. Structural and safety review

- The live adapter remains allowlisted to fixed command names only: `fortress-souls/get-dwarf-surroundings` and `fortress-souls/get-stock-summary`.
- `look_around` remains session-bound to the selected dwarf. The model does not choose unit ID, absolute coordinates, or z-level.
- The application still owns argument validation, authorization, cancellation, timeouts, output limits, schema validation, error mapping, receipts, and telemetry.
- Hidden-cell filtering is strict before the result reaches the model. The retained live hidden sample proves that hidden cells expose only `dx`, `dy`, and `visibility = hidden`.
- The bounded surroundings radius is now configurable by the caller within the reviewed v0.2 safety cap of `1..2`, without re-editing the Lua script.
- Live stock results remain exact-only. Approximate or bookkeeping-style counts remain absent from the product DTO.
- No generic DFHack command surface, raw observation persistence, write-capable path, streaming path, or unrelated framework capability was accepted.
